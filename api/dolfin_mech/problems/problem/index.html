dolfin_mech.problems.problem

Defines the Problem class.

Centralized manager for dolfin_mech simulations that synchronizes mesh topology, mixed-element function spaces, physical operators, and time-stepping logic to automate the assembly of nonlinear variational forms.

Module Contents

class dolfin_mech.problems.problem.Problem

Base class for managing Finite Element Problems in FEniCS.

This class acts as a container and manager for the essential components of a simulation: the mesh, the solution function spaces (unknowns), the operators (physics contributions), boundary conditions, and time stepping.

It abstracts the boilerplate code required to set up mixed-element formulations, manage input/output fields (FOIs/QOIs), and assemble the global residual and Jacobian forms for the nonlinear solver.

set_mesh(mesh, define_spatial_coordinates=True, define_facet_normals=False, compute_bbox=False, compute_local_cylindrical_basis=False)

Sets the computational domain and associated geometric quantities.

This method registers the mesh and initializes standard FEniCS measures (dx for volume, ds for surface). It can also compute spatial coordinates (reference X or spatial x) and local bases.

Parameters:

  • mesh – The dolfin Mesh object.

  • define_spatial_coordinates – If True, defines self.X (reference) or self.x (spatial) depending on the problem type (Inverse vs Forward).

  • define_facet_normals – If True, initializes self.mesh_normals.

  • compute_bbox – If True, computes the bounding box of the mesh.

  • compute_local_cylindrical_basis – If True, computes local radial (\(\mathbf{e}_R\)) and tangential (\(\mathbf{e}_T\)) vectors and a rotation matrix Q_expr.

set_measures(domains=None, boundaries=None, points=None)

Defines the integration measures for subdomains and boundaries.

Registers self.dV (volume measure), self.dS (surface measure), and self.dP (point measure), linking them to specific subdomain data if provided.

set_solution_finite_element()

Constructs the (potentially mixed) finite element for the global solution.

If multiple sub-solutions are defined (e.g., displacement and pressure), this creates a MixedElement.

set_solution_function_space(constrained_domain=None)

set_solution_functions()

Instantiates the actual FEniCS Functions for the solution.

Creates:

  • sol_func: The current solution vector \(\mathbf{u}_{n+1}\).

  • sol_old_func: The solution at the previous step \(\mathbf{u}_n\).

  • dsol_func: The increment (Newton update) \(\delta \mathbf{u}\).

  • dsol_test: The test function \(\delta \mathbf{v}\).

  • dsol_tria: The trial function \(\Delta \mathbf{u}\) (for the Jacobian).

It also splits these functions back into their sub-solution components for easy access by operators.

set_variational_formulation(k_step=None)

Assembles the global variational forms for the nonlinear solver.

This aggregates the residual forms from all active operators and computes the tangent Jacobian matrix using automatic differentiation.

\[\mathbf{R}(\mathbf{u}) = \sum \mathbf{R}_{op}(\mathbf{u}) \ \mathbf{J} = \frac{\partial \mathbf{R}}{\partial \mathbf{u}}\]
Parameters:

k_step – Optional step index to include step-specific operators.
