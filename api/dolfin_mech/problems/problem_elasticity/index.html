dolfin_mech.problems.problem_elasticity

Defines the ElasticityProblem class.

Specialized manager for linearized elastic simulations that automates the setup of infinitesimal kinematics, multi-material elasticity operators, and stable mixed displacement-pressure formulations for incompressible media.

Bases: dolfin_mech.problems.problem.Problem

Problem class for linearized elasticity simulations.

This class manages the lifecycle of a linear elastic simulation, including:

  • Mesh and measure initialization.

  • Finite element spaces for displacement and (optionally) pressure.

  • Linearized kinematics (infinitesimal strain).

  • Material operator assembly (e.g., Hooke’s Law).

  • Fields of Interest (FOI) and Quantities of Interest (QOI) for post-processing.

The problem can handle mixed formulations (u-p) for incompressible materials by setting w_incompressibility=True.

Variables:

  • w_incompressibility (bool) – If True, a pressure field is added to enforce incompressibility.

  • kinematics (LinearizedKinematics) – Object handling infinitesimal strain.

  • displacement_subsol (SubSolution) – Container for displacement fields.

  • pressure_subsol (SubSolution) – Container for pressure fields (if used).

add_displacement_subsol(degree)[source]

Adds the displacement vector sub-solution to the problem’s solution space.

The displacement field \(\mathbf{u}\) is defined using the Continuous Galerkin (CG) family, which ensures that the displacement is continuous across element boundaries (standard Lagrange elements).

In a displacement-based finite element formulation, the polynomial degree of this sub-solution determines the spatial resolution and accuracy of the strain and stress calculations:

  • Degree 1: Linear elements. Strains and stresses are constant within each element.

  • Degree 2: Quadratic elements. Strains and stresses vary linearly within each element, often providing better convergence for complex geometries and bending problems.

Parameters:

degree (int) – Polynomial degree for the displacement interpolation.

add_pressure_subsol(degree)[source]

Adds the pressure scalar sub-solution to the problem’s solution space.

The choice of finite element family depends on the requested polynomial degree:

  • Degree 0: Uses a Discontinuous Galerkin (DG) family. This

    results in a pressure field that is constant within each element but discontinuous across cell boundaries.

  • Degree > 0: Uses a Continuous Galerkin (CG) family (Standard

    Lagrange elements). This results in a pressure field that is continuous across the entire domain.

In mixed formulations, the choice of pressure space is critical for stability. For instance, a \(P_2-P_1\) (Taylor-Hood) pair uses continuous linear pressure, while a \(P_1-P_0\) pair uses discontinuous constant pressure.

Parameters:

degree (int) – Polynomial degree for the pressure interpolation.

set_subsols(displacement_degree=1, pressure_degree=None)[source]

Configures and initializes the solution fields (sub-solutions) for the problem.

This method defines the functional spaces for the unknowns being solved. It always initializes a displacement field \(\mathbf{u}\). If the problem is flagged for incompressibility (w_incompressibility=True), it also initializes a pressure field \(p\).

For incompressible or nearly-incompressible materials, the method defaults to a Taylor-Hood element pair (e.g., \(P_k / P_{k-1}\)) to satisfy the Inf-Sup (Babuska-Brezzi) stability condition, preventing spurious pressure oscillations.

Parameters:

  • displacement_degree (int) – Polynomial degree for the displacement Lagrange elements. Defaults to 1 (linear).

  • pressure_degree (int, optional) – Polynomial degree for the pressure elements. If None and incompressibility is enabled, it defaults to displacement_degree - 1.

Note

Common stable pairs include:

  • P2-P1: Quadratic displacement, Linear pressure.

  • P1-P0: Linear displacement, Constant pressure (requires specific stabilization).

set_quadrature_degree(quadrature_degree=None)[source]

Configures the numerical integration precision for the variational forms.

Quadrature (or Gauss) integration is used to evaluate the integrals in the residual and Jacobian. This method sets the polynomial degree that the quadrature rule should be able to integrate exactly.

The quadrature_degree can be set using the following options:

  • int: A specific degree (e.g., 2, 3).

  • “full”: Allows FEniCS/FFC to automatically determine the degree

    required for exact integration of the current UFL expressions.

  • “default”: Uses a heuristic based on the element type and polynomial

    degree of the displacement to find a balance between speed and precision:

    • For Simplex cells (Triangle/Tet): \(\max(2, 2(k-1))\)

    • For Hypercube cells (Quad/Hex): \(\max(2, 2(dim \cdot k-1))\)

    where \(k\) is the displacement degree.

Parameters:

quadrature_degree (int, str, or None) – The desired integration rule setting.

Raises:

AssertionError – If an invalid string or type is provided.

set_kinematics()[source]

Initializes the linearized kinematic quantities and registers default kinematic fields of interest.

This method instantiates a LinearizedKinematics object using the current and previous displacement sub-solutions. This object computes the infinitesimal strain tensor \(\epsilon\), defined as the symmetric part of the displacement gradient:

\[\epsilon = \frac{1}{2} (\nabla \mathbf{u} + \nabla \mathbf{u}^T)\]

Additionally, it registers the strain tensor as a Field of Interest (FOI), ensuring that \(\epsilon\) is available for spatial visualization and post-processing (e.g., in XDMF files).

get_subdomain_measure(subdomain_id=None)[source]

Returns the integration measure associated with a specific subdomain ID.

In finite element analysis, different material properties or constraints are often assigned to different regions of the mesh. This method provides the correct volume measure (\(dV\)) for a given region.

  • If subdomain_id is None, the measure represents the entire domain.

  • If subdomain_id is an integer, the measure is restricted to cells tagged with that specific value in the domain MeshFunction.

Parameters:

subdomain_id (int, optional) – The marker value identifying the subdomain. Corresponds to the values found in the domains_mf MeshFunction.

Returns:

A Dolfin measure object used in UFL variational forms.

Return type:

dolfin.Measure

add_elasticity_operator(material_model, material_parameters, subdomain_id=None)[source]

Adds a linearized elasticity operator to the variational problem.

This method defines the internal virtual work contribution for the solid skeleton. It associates a specific constitutive behavior (material model) with the displacement field through the infinitesimal strain tensor.

The contribution to the residual form is:

\[\delta \Pi_{int} = \int_{\Omega} \sigma(\epsilon) : \delta \epsilon \, d\Omega\]
where:

  • \(\sigma\) is the Cauchy stress tensor defined by the material_model.

  • \(\delta \epsilon\) is the virtual infinitesimal strain (symmetric part of the gradient of the test function).

Parameters:

  • material_model (str) – The name of the constitutive model to use (e.g., “Hooke”).

  • material_parameters (dict) – A dictionary containing the necessary coefficients for the chosen model (e.g., Young’s modulus ‘E’ and Poisson’s ratio ‘nu’).

  • subdomain_id (int, optional) – Optional ID to restrict the material behavior to a specific part of the mesh. If None, the operator is applied to the entire volume.

Returns:

The instantiated elasticity operator.

Return type:

dmech.LinearizedElasticityOperator

add_hydrostatic_pressure_operator(subdomain_id=None)[source]

Adds the hydrostatic pressure coupling term to the virtual work equation.

In a mixed \(\mathbf{u}-p\) formulation, the internal virtual work is augmented by a pressure term. This operator represents the work done by the hydrostatic pressure \(p\) against the virtual volumetric strain (the divergence of the test displacement \(\delta \mathbf{u}\)).

The contribution to the residual is:

\[\delta \Pi_{p} = - \int_{\Omega} p \cdot \text{div}(\delta \mathbf{u}) \, d\Omega\]

This term ensures that the pressure \(p\) acts as the mechanical hydrostatic stress component within the momentum equation, effectively balancing the external loads while satisfying the incompressibility constraint.

Parameters:

subdomain_id (int, optional) – Optional ID to restrict the pressure coupling to a specific subdomain.

Returns:

The instantiated hydrostatic pressure operator.

Return type:

dmech.LinearizedHydrostaticPressureOperator

add_incompressibility_operator(subdomain_id=None)[source]

Adds the incompressibility constraint equation to the variational formulation.

In a mixed \(\mathbf{u}-p\) formulation, this operator enforces the kinematic constraint that prevents volume change. For linearized elasticity, this corresponds to the weak form of the divergence-free condition:

\[\int_{\Omega} \text{div}(\mathbf{u}) \cdot \delta p \, d\Omega = 0\]
where:

  • \(\mathbf{u}\) is the displacement field.

  • \(\delta p\) is the pressure test function.

This constraint is essential when the Poisson’s ratio \(\nu\) approaches 0.5, as standard displacement-based elements suffer from volumetric locking.

Parameters:

subdomain_id (int, optional) – Optional ID to restrict the constraint to a specific subdomain. If None, it is applied to the entire domain.

Returns:

The instantiated incompressibility operator.

Return type:

dmech.LinearizedIncompressibilityOperator

add_elasticity_operators(elastic_behaviors)[source]

Orchestrates the addition of multiple material behaviors and mixed-field constraints.

This method processes a list of material definitions to build the problem’s total internal virtual work. For each behavior, it:

  1. Registers a LinearizedElasticityOperator.

  2. Automatically adds the resulting Cauchy stress \(\sigma\) to the Fields of Interest (FOI) for visualization.

If w_incompressibility was enabled during problem initialization, this method also appends the hydrostatic pressure coupling and the volumetric constraint (continuity equation) to the variational form.

Parameters:

elastic_behaviors (list[dict]) – A list of dictionaries, where each dictionary contains: - "model" (str): The material model name. - "parameters" (dict): Parameters for the model (e.g., E, nu). - "subdomain_id" (int, optional): The ID of the cell subdomain. - "suffix" (str, optional): A string to append to the stress FOI name.

add_global_strain_qois()[source]

Adds integral Quantities of Interest (QOIs) for all independent infinitesimal strain components.

This method extracts the strain tensor \(\mathbf{\epsilon}\) from the problem’s kinematics and defines a scalar QOI for each independent component by integrating it over the domain volume \(V\).

The resulting QOIs are named using the pattern e_ij (e.g., e_XX, e_XY). The integrated values correspond to:

\[E_{ij} = \int_{\Omega} \epsilon_{ij} \, d\Omega\]

The components added are determined by the spatial dimension of the mesh (\(d=1, 2, 3\)). For example, in 3D, all six independent components (three axial, three shear) are registered.

add_global_stress_qois()[source]

Adds integral Quantities of Interest (QOIs) for all independent Cauchy stress components.

This method iterates through all registered operators in the problem. For every operator that possesses a material attribute with a defined sigma (Cauchy stress), it accumulates the volume integral of each tensor component.

The resulting QOIs are named using the pattern s_ij (e.g., s_XX, s_XY). These values represent the total force-like contribution over the domain:

\[\Sigma_{ij} = \sum_{k} \int_{\Omega_k} \sigma_{ij}^{(k)} \, d\Omega_k\]

The number of components added depends on the spatial dimension of the problem (\(d=1, 2, 3\)).

add_global_pressure_qoi()[source]

Adds a global Quantity of Interest (QOI) for the integrated pressure field.

This method iterates through all registered operators in the problem and accumulates the contribution of their pressure fields \(p\). The resulting QOI is named “p” and represents the integral of the pressure over the relevant domains:

\[P_{global} = \sum_{i} \int_{\Omega_i} p_i \, d\Omega_i\]

This is particularly useful for monitoring the average internal pressure in incompressible or poromechanical simulations.

Note

For purely elastic operators where pressure is not an explicit unknown, the hydrostatic pressure can alternatively be calculated as \(p = -\frac{1}{3} \text{tr}(\sigma)\).
