dolfin_mech.materials.Material

Material Constitutive Laws Module.

This module provides a unified framework for defining and instantiating material constitutive laws within the dolfin_mech ecosystem. It serves two primary roles:

  1. Parameter Normalization: A base Material class provides automated conversion between engineering constants (e.g., \(E, \nu\)) and theoretical parameters (e.g., \(\lambda, \mu\)).

  2. The Material Factory: A robust dispatcher function that maps string identifiers to specific elastic material classes.

Crucial Material Factory Identifiers

The material_factory function supports an exhaustive list of aliases. These are categorized below by their underlying constitutive logic as depicted in the next table. Otherwise, you can use the Material name in lowercase.

Key Features

  • Lame Parameter Computation: Automated handling of Plane Stress (PS) vs. Plane Strain/3D conditions when calculating $lambda$ from Young’s Modulus and Poisson’s ratio.

  • Hyperelastic Coefficients: Built-in logic to derive hyperelastic constants (\(C_{10}, C_{01}\)) from shear moduli if they are not explicitly provided.

  • Decoupled Formulations: The _bar suffix in the factory triggers the decoup=True flag, essential for nearly-incompressible materials where volumetric and isochoric responses are separated.

Note

The factory uses case-insensitive matching for most identifiers, but standardizing on the aliases above is recommended for readability.

class dolfin_mech.materials.Material.Material

Base class for material models providing utility methods for parameter conversion.

This class handles the translation between common engineering constants (Young’s modulus \(E\), Poisson’s ratio \(\nu\)) and the theoretical parameters required by constitutive laws (Lamé constants \(\lambda, \mu\), bulk modulus \(K\), etc.).

get_lambda_from_parameters(parameters)

Compute the first Lamé parameter \(\lambda\).

Parameters:

parameters (dict) – Dictionary containing material constants. Expected keys: "lambda" OR ("E" AND "nu"). Optional key: "PS" (bool) for Plane Stress formulation.

Returns:

The value of \(\lambda\).

Return type:

dolfin.Constant

get_mu_from_parameters(parameters)

Compute the second Lamé parameter (shear modulus) \(\mu\).

Parameters:

parameters (dict) – Dictionary containing material constants. Expected keys: "mu" OR ("E" AND "nu").

Returns:

The value of \(\mu\).

Return type:

dolfin.Constant

get_lambda_and_mu_from_parameters(parameters)

Compute both Lamé parameters simultaneously.

Returns:

(dolfin.Constant, dolfin.Constant) representing (\(\lambda, \mu\)).

Return type:

tuple

get_K_from_parameters(parameters)

Compute the bulk modulus \(K\).

If "K" is not in parameters, it is derived from \(\lambda\) and \(\mu\) as \(K = (3\lambda + 2\mu)/3\).

get_G_from_parameters(parameters)

Compute the shear modulus \(G\). Equivalent to \(\mu\).

get_C0_from_parameters(parameters, decoup=False)

Compute the hyperelastic coefficient \(C_0\).

This parameter is typically associated with the volumetric or compressibility part of the strain energy density function.

Parameters:

  • parameters (dict) – Dictionary of material parameters.

  • decoup (bool, optional) – If True, computes \(C_0\) from the bulk modulus \(K\): \(C_0 = K/4\). If False, computes it from \(\lambda\): \(C_0 = \lambda/4\). Defaults to False.

Returns:

The computed \(C_0\) value.

Return type:

dolfin.Constant

get_C1_from_parameters(parameters)

Compute the hyperelastic coefficient \(C_1\).

In many models, \(C_1\) is related to the shear response of the material.

Parameters:

parameters (dict) – Dictionary of material parameters. Expected keys: "C1", "c1", "mu", or ("E" and "nu").

Returns:

The computed \(C_1\) value.

Return type:

dolfin.Constant

Notes

If not explicitly provided, it is derived as \(C_1 = \mu/2\).

get_C2_from_parameters(parameters)

Compute the hyperelastic coefficient \(C_2\).

Parameters:

parameters (dict) – Dictionary of material parameters. Expected keys: "C2", "c2", "mu", or ("E" and "nu").

Returns:

The computed \(C_2\) value.

Return type:

dolfin.Constant

Notes

If not explicitly provided, it is derived as \(C_2 = \mu/2\).

get_C1_and_C2_from_parameters(parameters)

Compute Mooney-Rivlin coefficients \(C_1\) and \(C_2\).

If only "mu" is provided, it assumes \(C_1 = C_2 = \mu/4\).

dolfin_mech.materials.Material.material_factory(kinematics, model, parameters)

Factory function to instantiate the appropriate material model class.

Parameters:

  • kinematics (Kinematics) – An instance of a Kinematics class.

  • model (str) – The name of the material model (e.g., "Hooke", "NeoHookean", "SVK").

  • parameters (dict) – Material parameters passed to the model constructor.

Returns:

An instance of the requested material model.

Return type:

Material

Example

>>> mat = material_factory(kin, "NH", {"E": 10.0, "nu": 0.3})
