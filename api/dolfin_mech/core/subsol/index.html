dolfin_mech.core.subsol

Defines the SubSol class.

Manages individual field variables (e.g., displacement, pressure) within a coupled system, including their finite element definitions and initialization strategies.

Module Contents

class dolfin_mech.core.subsol.SubSol(name, fe, init_val=None, init_fun=None)

Class representing a single unknown field (sub-solution) in a mixed problem.

A SubSol encapsulates the definition of a specific variable (e.g., displacement vector \(\mathbf{u}\), pressure scalar \(p\)) within a larger coupled system. It manages the Finite Element (FE) type and the strategy for initialization (constant value vs. spatial field).

This abstraction simplifies the assembly of mixed function spaces and the application of initial conditions in the Problem class.

Parameters:

  • name (str) – Name of the variable (e.g., “U”, “p”, “Phis”).

  • fe (dolfin.FiniteElement) – The FEniCS FiniteElement definition (e.g., VectorElement(“CG”, cell, 2)).

  • init_val (float or ndarray) – (Optional) A constant value (scalar or array) to initialize the field everywhere.

  • init_fun (dolfin.Function) – (Optional) A FEniCS Function or Expression to initialize the field spatially.

init_with_val()

Initializes the function with a constant value.

This method creates a constant FEniCS Expression from self.init_val and interpolates it into the function space. It updates both the current solution (self.func) and the previous step solution (self.func_old).

init_with_field()

Initializes the function using an existing field (Function or Expression).

This copies the vector values from self.init_fun directly into self.func and self.func_old. This is useful for restarting simulations or mapping results from a previous step.
