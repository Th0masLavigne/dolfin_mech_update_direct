dolfin_mech.core.TimeVaryingConstant

Utility for time-dependent FEniCS constants.

Module Contents

class dolfin_mech.core.TimeVaryingConstant.TimeVaryingConstant(val=None, val_ini=None, val_fin=None)

A wrapper for dolfin.Constant that evolves linearly over a time step.

This class manages a FEniCS constant whose value needs to change during the simulation (e.g., ramping up a load from 0 to 100 over a load step). It supports both scalar and vector constants.

Mechanism: The value at any normalized time step \(\tau \in [0, 1]\) is computed as:

\[v(\tau) = (1 - \tau) v_{ini} + \tau v_{fin}\]

It provides methods to update the underlying dolfin.Constant automatically when driven by a time integrator.

Parameters:

  • val – A single value (if the constant is fixed).

  • val_ini – The initial value at the start of the step.

  • val_fin – The final value at the end of the step.

set_value_sca(val)

Updates the value for a scalar constant.

set_value_vec(val)

Updates the value for a vector constant.

set_value_at_t_step(t_step)

Interpolates and sets the value based on the normalized step time.

Parameters:

t_step – Normalized time \(\tau \in [0,1]\).

set_dvalue_at_t_step(t_step)

Sets the value to the increment \(v(\tau) - v_{old}\).

This is useful for incremental formulations where the unknown is the increment.

restore_old_value()

Restores the internal state to the previous value.

Used when a time step fails and needs to be retried.

homogenize()

Sets the value to zero (homogeneous condition).
