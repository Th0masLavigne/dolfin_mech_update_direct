Main driver class for time-dependent finite element simulations.

The TimeIntegrator orchestrates the entire simulation workflow. It iterates through the sequence of defined Steps, and within each step, it advances time \(t\) from \(t_{ini}\) to \(t_{fin}\).

Core Responsibilities:

  1. Adaptive Time Stepping: Dynamically adjusts the time increment \(\Delta t\) based on solver performance.

    • Success: Increases \(\Delta t\) (acceleration) if convergence is fast.

    • Failure: Decreases \(\Delta t\) (deceleration/cutting) if convergence fails, retrying the step.

  2. State Management: Updates time-dependent operators and constraints before each solve.

  3. Output Management: Writes results to disk (VTU/XDMF for fields, DAT for QOIs, log files for monitoring).

Algorithm:

For each Step:
    t = t_ini
    While t < t_fin:
        Try solving for t + dt
        If Converged:
            Update solution and history variables
            Write output
            Increase dt (if iterations low)
            t = t + dt
        Else:
            Restore previous solution
            Decrease dt (cut time step)
            Retry
Parameters:

  • problem (Problem) – The problem instance containing physics and mesh.

  • solver (NonlinearSolver) – The nonlinear solver instance.

  • parameters (dict) – Dictionary controlling time stepping behavior: - n_iter_for_accel (int): Max iterations to trigger time step increase. - n_iter_for_decel (int): Min iterations to trigger time step decrease. - accel_coeff (float): Factor to increase dt by. - decel_coeff (float): Factor to decrease dt by.

  • print_out – Enable/disable main log file output.

  • print_sta – Enable/disable statistics table output (.sta file).

  • write_qois – Enable/disable writing Quantity of Interest data (.dat file).

  • write_sol – Enable/disable writing full field solution (.xdmf file).

  • write_vtus – Enable/disable writing VTU files for ParaView.

close()

Closes all open file handles (logs, tables, QOI data, XDMF).

integrate()

Executes the time integration loop.

This is the main entry point to run the simulation after setup.

Returns:

True if the simulation completed successfully, False otherwise.
