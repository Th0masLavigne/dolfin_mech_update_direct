dolfin_mech.core.loading

Defines the Loading base class.

Couples FEniCS integration measures with time-varying scalar magnitudes to handle evolving boundary tractions, pressures, or body forces.

Module Contents

class dolfin_mech.core.loading.Loading(measure, val=None, val_ini=None, val_fin=None, xyz_ini=None, N=None)

Base class for managing time-dependent mechanical loads.

This class serves as a container for loading parameters (such as pressure, traction, or body forces) that vary during a simulation. It associates a FEniCS measure with a dolfin_mech.TimeVaryingConstant.

Parameters:

  • measure (dolfin.Measure) – FEniCS integration measure.

  • val (float, optional) – A constant value for the entire simulation.

  • val_ini (float, optional) – Initial value for time-varying loading.

  • val_fin (float, optional) – Final value for time-varying loading.

  • xyz_ini (numpy.ndarray or list, optional) – Initial position vector.

  • N (numpy.ndarray or list, optional) – Directional vector (e.g., surface normal).

Notes

You must provide either (val) OR (val_ini and val_fin). The magnitude is stored internally within a dolfin_mech.TimeVaryingConstant.

Variables:

  • measure (dolfin.Measure) – The FEniCS measure (e.g., dx, ds) defining the integration domain for the load.

  • tv_val (dolfin_mech.TimeVaryingConstant) – Manager for the scalar load magnitude.

  • val (dolfin.Constant) – The current value of the load magnitude, linked to the underlying FEniCS constant.

  • N (dolfin.Constant, optional) – Normal vector or direction associated with the load.

  • xyz_ini (dolfin.Constant, optional) – Initial coordinates, often used for position-dependent loading in the reference configuration.
