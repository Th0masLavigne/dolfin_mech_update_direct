dolfin_mech.runs.heartslice_mesh

Defines the run_HeartSlice_Mesh utility.

Automates the generation of a 2D annular mesh representing a ventricular cross-section. It integrates Gmsh for triangulation and provides pre-tagged FEniCS MeshFunctions for endocardial and epicardial boundaries, as well as specific anatomical pinning points.

Module Contents

dolfin_mech.runs.heartslice_mesh.HeartSlice_Mesh(params: dict = {})[source]

Generates a 2D annular mesh representing a heart slice using Gmsh.

This function creates a mesh for a simplified 2D cross-section of a ventricle (an annulus). It handles geometry creation, meshing, conversion to FEniCS format, and the creation of boundary/point markers.

Geometry: Defined by a center \((X_0, Y_0)\), an inner radius \(R_i\) (endocardium), and an outer radius \(R_e\) (epicardium).

Markers:

  • Boundaries:

    • ID 1 (\(S_i\)): Inner boundary (Endocardium).

    • ID 2 (\(S_e\)): Outer boundary (Epicardium).

  • Points (on the inner boundary \(R_i\)):

    • ID 1: Right (\(0\) rad).

    • ID 2: Top (\(\pi/2\) rad).

    • ID 3: Left (\(\pi\) rad).

    • ID 4: Bottom (\(3\pi/2\) rad).

Parameters:

params – Dictionary of mesh parameters: - X0, Y0 (float): Center coordinates (default: 0.5, 0.5). - Ri (float): Inner radius (default: 0.2). - Re (float): Outer radius (default: 0.4). - l (float): Characteristic element size (default: 0.1). - mesh_filebasename (str): Output filename prefix (default: “mesh”).

Returns:

A tuple containing: - mesh (dolfin.Mesh): The generated mesh. - boundaries_mf (dolfin.MeshFunction): Boundary markers. - Si_id (int): ID for inner boundary (1). - Se_id (int): ID for outer boundary (2). - points_mf (dolfin.MeshFunction): Vertex markers. - x*_sd (dolfin.AutoSubDomain): Subdomains for the 4 cardinal points on the inner ring.
