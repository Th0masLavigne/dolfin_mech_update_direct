dolfin_mech.runs.rivlincube_porohyperelasticity

Defines the RivlinCube_PoroHyperelasticity driver.

High-level manager for biphasic finite strain simulations. It orchestrates forward and inverse poro- hyperelastic problems on structured domains, coupling solid skeleton mechanics with fluid volume fraction evolution and pressure-balancing gravity loads for biomechanical validation.

dolfin_mech.runs.rivlincube_porohyperelasticity.RivlinCube_PoroHyperelasticity(dim=3, inverse=0, cube_params={}, porosity_params={}, move_params={}, mat_params={}, step_params={}, load_params={}, inertia_params={'applied': False}, res_basename='run_RivlinCube_PoroHyperelasticity', plot_curves=False, get_results=0, verbose=0)[source]

Runs the Rivlin Cube benchmark for Poro-Hyperelasticity (Forward or Inverse).

This function simulates a porous unit cube (or square) undergoing large deformations. It validates the coupled mechanics of the solid skeleton and the pore fluid.

Problem Types:

  • Forward (`inverse=0`): Given initial porosity \(\Phi_{s0}\) and loads, solve for displacement \(\mathbf{u}\) and current porosity \(\phi_s\).

  • Inverse (`inverse=1`): Given current porosity \(\phi_s\) and loads, solve for the reference configuration (displacement \(\mathbf{u}\)) and reference porosity \(\Phi_{s0}\).

Loading Scenarios (`load_params[“type”]`):

  • "internal": Applies internal pore pressure \(p_f\).

  • "external": Applies external traction \(P\) to specific boundaries.

  • "p_boundary_condition": Applies a pressure-balancing gravity load, often used for lung/organ mechanics validation.

Porosity Initialization (`porosity_params`): Can be initialized as a constant, from a file, or via random generation using various keys like "constant", "from_file", "mesh_function_xml", etc.

Parameters:

  • dim – Spatial dimension (2 or 3).

  • inverse – If 1, solves the inverse problem; else forward.

  • cube_params – Mesh generation parameters (or path to existing mesh).

  • porosity_params – Configuration for the initial porosity field.

  • move_params – Parameters for ALE mesh movement before solving.

  • mat_params – Material parameters for skeleton, bulk, and pore behavior.

  • step_params – Time-stepping parameters (Deltat, dt_ini).

  • load_params – Loading configuration (type and magnitude).

  • inertia_params – Dynamic simulation parameters (density rho).

  • res_basename – Output filename prefix.

  • plot_curves – If True, generates Matplotlib plots of QOIs vs. time/pressure.

  • get_results – If True, returns displacement and porosity fields.

  • verbose – Verbosity level.

Returns:

(Optional) Tuple (displacement_func, porosity_array, measure).
